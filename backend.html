02 ① サービス側API（プラットフォーム内完結）

ユーザーのClaude/GitHub連携を不要にし、サービス側でAPIを用意。ブラウザ内で受け取り→実行→成果物までを完結させる。

🔌
エージェント実行API
Platform-completed Agent Runtime
リファレンス実装あり
役割
フロントからの「このエージェントを使う」を受け、Claudeでエージェント定義（入力→出力）を実行して成果物を返す。連携ワークフローは複数エージェントを直列実行。
実装

認証
ログイン＋発行APIキー。電話番号認証で無料乱用を防止（会議提案）。プランで利用回数・機能を制御。
課金判定
標準（月5本等の上限）／プレミアム（全機能）をAPI側で判定し、上限超過はアップセル誘導。
コスト
APIコストはサービス側負担。1実行あたりのClaude費は小さく（後述）、月額に十分収まる見通し。

03 ② 情報源の取り込みパイプライン

海外の最新記事・事例、書籍PDF＋OCR、PR TIMESの過去プレスをAIに読ませ、「即実行できる仕組み」と時事ニュースの供給源にする。

取得→正規化（PDF/OCR・本文抽出）→Claudeで要約・構造化
（重複しない制約）→索引・保存→辞書 / ニュースへ供給
情報源取得方法用途
海外の最新記事・事例Web検索API（Tavily）／RSS で収集時事ニュース・新スキルの種
書籍（PDF）PDF化 → OCR（テキスト化）戦略立案・ノウハウの学習素材
PR TIMES 過去プレススクレイピング（規約順守・レート制御）成功事例の分析・施策の裏付け
◆ 「重複しない」を仕組みで担保
取り込み時に既出コンテンツと突合し、過去に出した情報と重複しないものだけを要約・投稿する（プロトタイプのニュース自動フィードで実装済みのロジックを、本番はDB照合で恒久化）。情報の鮮度＝解約理由をなくす最大の価値。

リファレンス: backend/pipeline/ingest.py（取得→要約→重複チェック→保存）を同梱。

06 コスト試算

サービス側APIコスト（Claude）は1実行あたり小さく、月額（4,000〜9,800円 ≒ $27〜$65）に十分収まる。

コスト源単価目安月額への効き
エージェント1実行（Claude）約 $0.05〜0.30小さい
標準プラン（月5本想定）約 $0.3〜1.5／人月粗利大（対 ¥4,000〜5,000）
プレミアム（多用・100本想定）約 $5〜30／人月要監視（対 ¥9,800）
取り込みパイプライン検索=ほぼ無料・OCR/計算=小小さい
ニュース要約（Claude）1件 数円〜小さい
◆ 結論: 粗利は健全、ただしヘビーユーザーは要監視
標準プランはClaude費が月$1前後で粗利が大きい。プレミアムの使い放題は上限（回数・DL）とモデルの出し分け（定型=Haikuで安く）でコストを抑える設計が必須。会議の「1人あたりAPIコストは高額にならない」見通しと整合。

07 リスク・未決 と 段階導入

  • API乱用・原価: 電話番号認証＋利用上限＋モデル出し分けで、プレミアム使い放題の原価を制御。
  • スクレイピングの規約: PR TIMES等は利用規約・レート制御・robots順守。取得可否を事前確認。
  • 人間テストの体制: 品質はAIで自動判定できない。直氏・小林氏のテスト工程を運用に固定（単純労働では代替不可）。
  • スケール: 同時実行数・レート制限（Claude側／自社側）を監視し、キューイングで平準化。
