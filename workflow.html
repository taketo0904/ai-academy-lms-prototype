Workflow Design · 設計書

AI動画コンテンツ 製造ワークフロー

Web上のAI情報を調査し、企画→台本→素材生成→紐付け→動画生成→編集→説明文までを8体のエージェントで連結する、月20本規模の量産パイプライン。さらに最上位に、この8体を自動で構築・保守するビルダーエージェント（メタ層）を置く。使用APIと概算コストを含む設計仕様。

エージェント: 8体 ＋ ビルダー 前提: 月20本以上 動画: Higgsfield AI（公式API） 調査: Tavily LLM: Claude 更新: 2026-07-09

01 全体像

最上位のビルダーが下の8体を構築・保守し、実行層の8エージェントが直列で1本を作る。動画生成には、生成素材に加えてこちらから任意の素材・画像を渡せる。人が挟まるのは「企画GO」と「最終レビュー」の2点。

⚙️ メタ層 ── ビルダーエージェント（この8体を作る／動かせるようにするエージェント）

この設計書を入力に、下の8体を「動く実体」として自動生成・配線し、仕様変更時は再生成・更新まで担う。人は要件と承認のみ。

↓ 生成・配線・保守
1
🔍 AI情報 調査
X/Web上の最新AIトレンド・話題を収集し、要点を構造化。
出力 トレンド要約（トピック・要点・出典）
2
📝 動画企画 人: GO判断
調査結果から動画の企画案（狙い・ターゲット・尺・構成）をテキスト書式で生成。
出力 企画書（構成・尺・トンマナ）
3
📄 台本作成 新規
企画を受け、シーンごとのナレーション原稿・テロップ・カット割りを含む台本を作成。
入力 企画書 出力 台本（シーン単位・機械可読）
4
🎨 素材生成 新規
台本の各シーンに必要な画像・図・アイキャッチ等の素材を生成。
入力 台本 出力 素材ファイル群（＋メタ情報）
5
🔗 素材×台本 紐付け 新規
生成素材とこちら提供素材を、台本のシーンに割り当て、動画生成用のショットリストを組む。
入力 台本＋素材 出力 ショットリスト（シーン↔素材↔尺）
6
🎬 動画生成 こちら提供の素材/画像も入力
ショットリストをHiggsfieldプロンプトに変換し、クリップを生成。任意でこちらの素材・画像をimage-to-videoの入力に使う。
入力 ショットリスト＋（任意）手動素材 出力 動画クリップ群
7
✂️ 動画編集 新規
クリップを台本順に結合し、テロップ・BGM・トランジション・尺調整を適用して1本に仕上げる。
入力 クリップ群＋台本 出力 完成動画
8
🗒️ 説明文生成 人: 最終レビュー
完成動画と企画書から、タイトル・説明文・SNS投稿文・字幕を生成。
入力 完成動画＋企画書 出力 説明文一式
◆ オーケストレーションの要点
段間の整合性を保つため、台本を機械可読な定形フォーマット（シーンID・尺・ナレーション・必要素材）で持たせる。台本がAgent 4〜7すべての共通ソースになり、「素材↔シーン↔クリップ」がIDで一意に対応するのが肝。動画生成と素材生成は非同期・失敗リトライ前提で組む。

02 各エージェント仕様

役割・入出力・使用API/モデル。モデルは難易度で出し分ける（企画/台本=高性能、紐付け/整形=低コスト）。最上位にビルダー（メタ層）、その下に実行層の8体。

⚙️
Agent ⚙ ── ビルダー（構築エージェント）メタ層
Pipeline Builder / Orchestrator-Constructor
役割
この設計書を入力に、Agent 1〜8の実体（各システムプロンプト・API連携コード・段間スキーマ・オーケストレーション・リトライ）を生成し、動く状態に組み上げる。仕様変更時は差分を再生成・更新。
入力
設計書（8体の定義・I/Oスキーマ・API割当）、資格情報（各APIキー）、変更要件
出力
実行可能なパイプライン一式（各エージェントのコード/プロンプト・実行スクリプト・設定・テスト）
API
Claude claude-opus-4-8 によるエージェンティックなコード生成。実装基盤は Claude Agent SDK または Managed Agents（コードを書く→動かす→直すループ、bash・ファイル操作・検証を回す）
備考
いわゆる「エージェントを作るエージェント」。初期構築と仕様変更の反映を自動化。ビルド時のみ動くためper-videoコストには乗らない（第4章の注記）
🔍
Agent 1 ── AI情報 調査
Research / Trend Scan
役割
指定テーマで X・ニュース・技術ブログ等から直近の話題を収集・要約・分類。
入力
調査テーマ、対象期間、除外条件
出力
トレンド要約（トピック・要点・出典URL・話題度）
API
Tavily API（無料枠1,000cr/月）＋ Claude claude-sonnet-5
📝
Agent 2 ── 動画企画 立案
Content Planning（テキスト書式）
役割
調査要約から企画書（タイトル案・狙い・ターゲット・全体尺・構成）を生成。
入力
トレンド要約、フォーマット指定
出力
企画書（Markdown＋構造化）
API
Claude claude-opus-4-8（企画の質が全体を左右するため最上位）
備考
ここで人の「GO/差し戻し」を1回挟む（下流が高コストなため手戻り防止）
📄
Agent 3 ── 台本作成 新規
Script Writing
役割
企画を受け、シーンごとのナレーション原稿・テロップ文・カット割り・必要素材リストを含む台本を作成。
入力
Agent 2の企画書
出力
台本（シーンJSON）：各シーンに ID・尺・ナレーション・テロップ・「必要素材の記述」。以降の共通ソース
API
Claude claude-sonnet-5（構造化出力でシーンJSONを保証）
備考
ここで「シーン単位の必要素材」を明文化するのが、Agent 4/5を機械的に回すための鍵
🎨
Agent 4 ── 素材生成 新規
Asset Generation
役割
台本の「必要素材」記述から、シーンごとの画像・図・アイキャッチ等を生成。
入力
台本のシーンJSON（必要素材の記述）
出力
素材ファイル群（シーンIDでタグ付け）＋メタ情報
API
Higgsfield の画像モデル（text-to-image）or 別画像生成API。プロンプト整形は claude-haiku-4-5
備考
画像はクリップ動画より安価。生成素材＝image-to-videoの入力にもなる（Agent 6へ渡る）
🔗
Agent 5 ── 素材×台本 紐付け 新規
Asset ↔ Script Mapping
役割
生成素材＋こちら提供素材を台本の各シーンに割り当て、動画生成用のショットリストを組む。過不足（素材が足りないシーン）も検出。
入力
台本＋素材群（生成分＋手動提供分）
出力
ショットリスト：シーン↔素材↔尺↔生成プロンプトの対応表
API
Claude claude-haiku-4-5（対応付けロジック中心・低コスト）。素材の内容確認が要る場合は claude-sonnet-5 でビジョン照合
備考
IDベースで一意対応させ、動画生成が「どのシーンにどの素材で何秒」を迷わない状態にする
🎬
Agent 6 ── 動画生成
Video Generation（Higgsfield）
役割
ショットリストをHiggsfieldプロンプトに変換し、クリップを生成。
入力
ショットリスト＋（任意）こちら提供の素材・画像（image-to-videoの起点に使用）
出力
動画クリップ群
API
Higgsfield AI ── 公式REST API＋Python SDK。8秒クリップ ≈ 22クレジット〜。整形は claude-haiku-4-5
備考
手動素材の受け渡し口を明示的に用意（アップロード→シーン指定）。音声・リップシンクはHiggsfield内蔵で対応可
✂️
Agent 7 ── 動画編集 新規
Video Editing / Assembly
役割
クリップを台本順に結合し、テロップ・BGM・トランジション・尺調整・ナレーション同期を適用して1本に仕上げる。
入力
クリップ群＋台本（テロップ/尺/BGM指定）
出力
完成動画（＋書き出し設定）
API
Claude claude-sonnet-5 が編集指示（EDL＝カット/字幕/BGM/尺）を生成 → ffmpeg 等で自動結合・書き出し
備考
ffmpegベースなら追加API料金ほぼゼロ（自前計算）。AI編集APIも選択肢だが、まずはffmpegで確実に。凝った演出は人の最終調整で
🗒️
Agent 8 ── 説明文生成
Copy / Description
役割
完成動画と企画書から、タイトル・説明文・SNS投稿文・字幕/概要を生成。
入力
完成動画（尺・構成）＋企画書
出力
説明文一式（媒体別に複数パターン）
API
Claude claude-sonnet-5
備考

04 コスト試算

前提: 月20本。新規4体を足しても、費用構造は変わらず「動画生成が支配的」。

4-1. 単価の実態（1本あたり）

コスト源単価／本効き
Claude（6エージェント合計）約 $0.43小さい
Tavily（調査）実質 $0無視できる
素材生成（画像 約8枚・推定）約 $0.8小さい
動画編集（ffmpeg・自前）約 $0無視できる
Higgsfield 映像（Veo 3 Fast）約 $0.10／秒支配的
Higgsfield 映像（Premium 1080p）約 $0.26／秒支配的（Fastの約2.6倍）

※ 映像以外の固定費（Claude＋素材＋調査＋編集）＝約 $1.2／本。Claude内訳: 企画(Opus4.8)$0.17＋台本(Sonnet5)$0.05＋説明(Sonnet5)$0.04＋調査(Sonnet5)$0.08＋紐付け/整形(Haiku)$0.04＋編集指示(Sonnet5)$0.05。素材の画像クレジット単価は未確認のため推定。

◆ 結論は不変: コストは「動画の尺」でほぼ決まる
エージェントを4→8体に増やしても、追加分（台本・素材・紐付け・編集）は合計 約$0.9/本で誤差レベル。費用の9割超はいまも動画生成で、尺にほぼ比例する。設計判断は依然「1本を何秒にするか」が最重要。
◆ ビルダー（メタ層）のコストは別枠・ビルド時のみ
ビルダーは毎本走らせるものではなく、初期構築と仕様変更のときだけ動く。1回のビルド/大改修で Claude のエージェンティックコーディング（数百K〜数M tokens）＝$数〜数十程度。仕様変更が無い月は $0。したがって上のper-video試算には含めない。

◆ 尺と料金プラン（月20本）
ショート30秒×20本 ≈ 1,760クレジット/月なら Ultra 1契約（3,000cr・年払い$99/月）で収まる。10分×20本 ≈ 33,000クレジット/月だと Ultra＋大量トップアップで月$1,000超。長尺量産は「話すアバター（InfiniteTalk・長尺対応）」で1本を数クリップに畳む方が有利な可能性が高い（アバター系の正確な単価は要実測）。

05 リスク・未確定論点

✓ 解消: 動画生成の自動化可否
Higgsfieldに公式REST API＋公式Python SDKがあることを確認。ブラウザ自動化に頼らず8段を完全自動化できる。音声・リップシンクも内蔵。→ リスクは「実装可否」から「運用コスト」へ移った。
▲ 最大リスク: 動画尺に比例するコスト
費用の9割超が動画生成で、尺にほぼ比例。10分尺×月20本は月$1,000〜1,600規模に達しうる。「1本を何秒にするか」「長尺はアバターで畳めるか」が採算を直接決める。まずショート尺で単価・品質を検証し、採算が見えてから長尺へ、が安全。
  • 段間の整合性: 8段に増えたぶん、素材↔シーン↔クリップのID対応が崩れると破綻する。台本を「唯一の定形ソース」にする設計が必須。
  • 素材の重複/過不足: 素材生成とこちら提供素材が競合・不足しないよう、Agent 5で過不足検出を必ず入れる。
  • 編集の限界: ffmpeg自動編集は「結合・字幕・BGM・尺調整」までが確実圏。凝った演出・テンポは人の最終調整前提。
  • 品質のばらつき: 生成映像・画像は当たり外れがあり、リトライ＝追加クレジット前提。歩留まり+20〜40%を見込む。
  • API仕様の未確定点: Higgsfieldの音声/リップシンク・画像生成がAPI経由で叩けるか、画像クレジット単価は実装前に要検証。

06 段階導入案

1
上流の自動化（Agent 1〜3・8）
調査→企画→台本→説明文を自動化し、素材・生成・編集は手動。最小コストで「企画と台本の質」を検証。
2
素材と生成の接続（Agent 4〜6）
素材生成・紐付け・Higgsfield生成をショート尺で接続。手動素材の受け渡し口と、1本あたり実コスト・歩留まりを実測。
3
編集の自動化＋全自動運用（Agent 7）
ffmpeg編集を接続し8段を直列で自動実行。企画GOと最終納品前に指揮官レビューを挟む運用に。採算が見えたら尺・本数をスケール。
