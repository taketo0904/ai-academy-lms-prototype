Workflow Design · 設計書

AI動画コンテンツ 製造ワークフロー

Xを含むWeb上のAI情報を調査し、動画企画→動画生成→説明文までを4体のエージェントで連結する、月20本規模の量産パイプライン。使用APIと概算コストを含む設計仕様。

前提: 月20本以上 動画: Higgsfield AI 調査: Web検索（低コスト代替） LLM: Claude 作成: 2026-07-09

01 全体像

4エージェントを直列につなぎ、各段の出力が次段の入力になる。人が挟まるのは「企画GO判断」と「最終レビュー」の2点のみ。

AGENT 1
🔍
AI情報 調査
X/Web上の最新AIトレンド・話題を収集し、要点を構造化。
入力 テーマ/期間
出力 トレンド要約（構造化テキスト）
→
AGENT 2
📝
動画企画 立案
調査結果から動画コンテンツの企画案をテキスト書式で生成。
入力 トレンド要約
出力 企画書（構成・尺・シーン）
→
AGENT 3
🎬
動画 生成
企画のシーン記述をプロンプト化し、Higgsfieldで映像を生成。
入力 シーン記述
出力 動画クリップ
→
AGENT 4
🗒️
説明文 生成
完成動画のタイトル・説明文・投稿文・字幕を生成。
入力 企画＋動画
出力 説明文一式
◆ オーケストレーションの考え方
各エージェントは独立したステップとして呼び出し、前段の構造化出力（JSON/Markdown）を次段の入力に渡す。段間の整合性を保つため、企画書は機械可読な定形フォーマット（シーンごとにID・尺・プロンプト・ナレーション）で持たせるのが要。動画生成だけは非同期・失敗リトライ前提で組む。

02 各エージェント仕様

役割・入出力・使用API/モデルの詳細。モデルは難易度に応じて出し分ける（調査/企画は高性能、定型生成は低コスト）。

🔍
Agent 1 ── AI情報 調査エージェント
Research / Trend Scan
役割
指定テーマ（例:「建築×AI」「生成AIの最新動向」）で、X・ニュース・技術ブログ等から直近の話題を収集し、企画に使える粒度に要約・分類する。
入力
調査テーマ、対象期間（例: 直近7日）、除外条件
出力
トレンド要約（トピック名・要点・出典URL・話題度）を構造化テキストで
API
Tavily API（無料枠1,000クレジット/月・超過$0.008/クレジット）＋ Claude claude-sonnet-5 で要約・構造化。Claudeの web_search サーバーツールでAPI一本化も可
備考
Xの生投稿が必須でないため公式X API（月$200〜）は不要。Tavilyならこの用途は実質無料枠内
📝
Agent 2 ── 動画企画 立案エージェント
Content Planning（テキスト書式出力）
役割
調査要約を受け、動画1本の企画書を生成。タイトル案・狙い・ターゲット・全体尺・シーン割り（各シーンの秒数・映像プロンプト・ナレーション原稿）まで落とす。
入力
Agent 1のトレンド要約、フォーマット指定（尺・トンマナ・本数）
出力
企画書（Markdown＋機械可読なシーンJSON）。これがAgent 3/4の共通ソースになる
API
Claude claude-opus-4-8（企画の質が全体を左右するため最上位モデル）。構造化出力で定形JSONを保証
備考
ここで人の「GO/差し戻し」判断を1回挟むのが安全（動画生成は高コストなため手戻り防止）
🎬
Agent 3 ── 動画生成エージェント
Video Generation（Higgsfield）
役割
企画書のシーン記述を Higgsfield 向けプロンプトに変換し、クリップを生成。シーンを繋いで1本に構成。
入力
シーンJSON（プロンプト・尺・参照画像）
出力
動画クリップ（→結合して完成動画）
API
Higgsfield AI ── 公式REST API＋公式Python SDK あり（docs.higgsfield.ai / higgsfield-client）。クレジット制（8秒クリップ ≈ 22クレジット〜）。プロンプト整形は Claude claude-haiku-4-5
備考
公式APIで完全自動化が可能（ブラウザ自動化不要）。音声・リップシンク（Speak/Lipsync）内蔵で、話す動画なら音声もHiggsfield側で完結。1クリップ5〜8秒が基本＝長尺は結合前提
🗒️
Agent 4 ── 説明文生成エージェント
Copy / Description
役割
完成動画と企画書から、公開用のタイトル・説明文・SNS投稿文・字幕/概要を生成。
入力
企画書＋動画（尺・シーン構成）
出力
説明文一式（媒体別に複数パターン）
API
Claude claude-sonnet-5（定型コピー生成、コストと質のバランス）
備考
ナレーション音声が要る場合、話す動画はHiggsfield内蔵音声で完結。B-roll＋別ナレーションなら ElevenLabs（$0.10/1,000字・日本語最高品質）or Google Neural2（月100万字無料）を併用

04 コスト試算

前提: 月20本、1本あたりの各API消費を積み上げる。Claude料金は確定済み、Higgsfield・Web検索・TTSは調査結果で確定させる。

4-1. 単価の実態

コスト源単価本ワークフローでの効き
Claude（4エージェント合計）約 $0.30／本ごく小さい
Tavily（調査）実質 $0（無料枠内）無視できる
Higgsfield 映像（Veo 3 Fast）約 $0.10／秒支配的
Higgsfield 映像（Premium 1080p）約 $0.26／秒支配的（Fastの約2.6倍）
ElevenLabs（別ナレーション・任意）約 $0.2〜0.3／本小さい

※ クレジット単価は Ultra年払い ≈ $0.033／トップアップ ≈ $0.05。1クリップ8秒＝22クレジット(Fast)を基準に $0.035／クレジットで算出。Claude内訳: 調査(Sonnet5)$0.08＋企画(Opus4.8)$0.17＋整形(Haiku)$0.02＋説明(Sonnet5)$0.04。

◆ 結論: コストは「動画の尺」でほぼ決まる
Claude・調査・音声は合わせても1本$0.5前後で誤差レベル。費用の9割以上が動画生成で、しかも尺にほぼ比例する。つまり設計判断は「1本を何秒にするか」が最重要。生成AI映像は5〜8秒クリップの積み上げなので、10分尺は約75クリップ＝高額になる。

4-2. 尺別シナリオ（月20本・Veo 3 Fast品質）

動画の尺8秒クリップ数映像費／本本単価
(+Claude+調査)		月20本 合計
ショート（30秒）4$3.1約 $3.4約 $68

※ 上表はリトライ（作り直し）を含まない基準値。生成映像は当たり外れがあるため、歩留まりで実際は +20〜40%を見込む（例: 10分尺・月20本なら実質 $1,400〜1,600/月）。Premium画質は各費用が約2.6倍。

◆ 尺と料金プランの対応（月20本）
月の必要クレジットで契約プランが決まる。ショート30秒×20本 ≈ 1,760クレジット/月なら Ultra 1契約（3,000cr・年払い$99/月）で収まる。10分×20本 ≈ 33,000クレジット/月だと Ultra＋大量トップアップが必要で、月$1,000超。長尺を量産するなら「話すアバター（InfiniteTalk・長尺対応）」で1本を1〜数クリップに畳む方がクリップ数＝コストを抑えられる可能性が高い（※アバター系の正確なクレジット単価は要実測）。

05 リスク・未確定論点

✓ 解消: Higgsfieldの自動化可否
当初の最大懸念だった「動画生成段の自動化」は、Higgsfieldに公式REST API＋公式Python SDKがあることを確認。ブラウザ自動化に頼らず4段を完全自動化できる。音声・リップシンクもAPI/内蔵機能でカバー。→ リスクは「実装可否」から「運用コスト」に移った。
▲ 最大リスク: 動画尺に比例するコスト
費用の9割超が動画生成で、尺にほぼ比例する。10分尺×月20本は月$1,000〜1,600規模に達しうる（第4章）。「1本を何秒にするか」「長尺は話すアバターで畳めるか」が事業採算を直接決める。まずショート尺で単価と品質を検証し、採算が見えてから長尺へ、が安全。
  • 尺の制約: 生成AI映像は1クリップ5〜8秒。長尺は複数クリップの結合前提で、シーン割り設計とクリップ数管理が要。
  • 品質のばらつき: 生成映像は当たり外れがあり、リトライ＝追加クレジット前提。歩留まり+20〜40%を見込む。
  • クレジット失効: 月次クレジットは繰越なし、トップアップは約90日で失効。作る本数と契約プランを合わせないと無駄が出る。
  • API仕様の未確定点: 音声/リップシンク機能がAPI経由で叩けるかはSDK文書で未確認。実装前に要検証（画像・動画生成は文書化済み）。

06 段階導入案

1
半自動MVP（Agent 1・2・4を自動化）
調査→企画→説明文までを自動化し、動画生成だけ手動（Higgsfield手打ち）。最小コストで「企画の質」を検証。
2
動画生成の接続（Higgsfield公式SDK）
ショート尺で Agent 3 を接続し、1本あたりの実コストと生成品質・歩留まりを実測。API経由の音声/リップシンク可否もここで検証。
3
全自動＋レビューゲート
4段を直列で自動実行。企画GOと最終納品前に⚔️指揮官レビュー（GO/差し戻し）を挟む運用に。採算が見えたら尺・本数をスケール。
